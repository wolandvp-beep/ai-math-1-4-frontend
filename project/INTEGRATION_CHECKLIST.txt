Решайка — integration checklist

1. В src/config/env.js поставить реальный API_BASE_URL
2. Поднять backend endpoints по контрактам из папки contracts/
3. Проверить:
   - login
   - register
   - recover
   - logout
   - fetch profile
   - update profile
   - fetch subscription
   - restore purchase
4. Убедиться, что backend возвращает:
   - token
   - user.name
   - user.email
   - user.childName
   - subscription.status
   - subscription.plan
   - subscription.source
5. После этого подключать store billing
6. Потом делать второй язык

Итог:
frontend-каркас уже подготовлен, дальше основной риск не во фронте, а в серверных контрактах и биллинге.
