Решайка — внешний платёжный маршрут

Как теперь устроено:
1. Пользователь входит в аккаунт
2. В приложении открывает экран "Оплата на сайте"
3. Приложение просит backend создать web-session
4. Backend возвращает checkoutUrl
5. Пользователь оплачивает на внешнем сайте
6. Возвращается в приложение
7. Приложение обновляет доступ через /billing/access-status
8. Сервер сообщает, активен ли доступ

Что это даёт:
- приложение можно распространять через магазины
- основная оплата живёт снаружи
- доступ в приложении определяется сервером
- store billing остаётся только как запасной/будущий путь

Что дальше:
- подставить реальные backend endpoints
- связать внешний checkout с вашим сайтом оплаты
- настроить возврат в приложение после оплаты
- после этого можно начинать серверное тестирование доступа
